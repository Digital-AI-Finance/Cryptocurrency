#### Task 3: Create Glossary Infrastructure

**New file:** `D:/Joerg/Research/slides/cryptocurrency/glossary.md`

**Terms to Define (from all 4 lessons):**

**Lesson 1 - Blockchain Fundamentals:**
| Term | Definition | Example |
|------|------------|---------|
| Blockchain | A distributed, immutable ledger that records transactions across a network | Bitcoin's blockchain stores all BTC transactions since 2009 |
| Block | A container of transaction data with a header and body | Each Bitcoin block contains ~2000 transactions |
| Hash | A fixed-length fingerprint generated from variable-length data | SHA-256("Hello") = "2cf24dba..." |
| Merkle Tree | A tree structure where each leaf is a hash and each node is the hash of its children | Used in Bitcoin to summarize all transactions in one root hash |
| Consensus | Agreement among network participants on the valid state | Bitcoin uses Proof of Work for consensus |
| Proof of Work | A consensus mechanism requiring computational work to add blocks | Bitcoin miners solve puzzles requiring ~10 minutes of work |
| Proof of Stake | A consensus mechanism where validators stake tokens to participate | Ethereum 2.0 requires 32 ETH to become a validator |
| Genesis Block | The first block in a blockchain, hardcoded with no previous hash | Bitcoin's genesis block was mined Jan 3, 2009 |
| Nonce | A number used once, incremented during mining to find valid hash | Miners try billions of nonces to find one producing valid hash |
| Double-spending | Attempting to spend the same digital money twice | Alice tries to send 10 BTC to both Bob and Charlie |
| Node | A computer running blockchain software that validates transactions | Full nodes store entire blockchain history |
| 51% Attack | When an entity controls majority of network power to manipulate chain | Requires controlling >50% of all mining power |

**Lesson 2 - Cryptography:**
| Term | Definition | Example |
|------|------------|---------|
| SHA-256 | A cryptographic hash function producing 256-bit output | SHA-256("bitcoin") = "6b88c087..." |
| Public Key | The shareable part of a key pair used to receive funds | Your Bitcoin address is derived from your public key |
| Private Key | The secret part of a key pair used to sign transactions | A 256-bit random number that must never be shared |
| Digital Signature | Cryptographic proof that the private key holder approved a message | ECDSA signature proves Alice authorized a transfer |
| ECDSA | Elliptic Curve Digital Signature Algorithm | Bitcoin and Ethereum use secp256k1 curve |
| Wallet | Software managing private keys and creating transactions | MetaMask, Ledger hardware wallet |
| Seed Phrase | 12-24 words that can regenerate all your private keys | "abandon ability able about above absent..." |
| Address | A human-readable identifier derived from public key | 0x742d35Cc6634C0532925a3b844Bc9e7595f... |

**Lesson 3 - Ethereum:**
| Term | Definition | Example |
|------|------------|---------|
| Smart Contract | Self-executing code stored on blockchain | ERC-20 token contract with transfer(), balanceOf() |
| EVM | Ethereum Virtual Machine - runtime for smart contracts | Executes bytecode on every Ethereum node |
| Gas | Unit measuring computational effort on Ethereum | A simple transfer costs 21,000 gas |
| Gas Price | Amount of ETH paid per unit of gas | 50 Gwei = 0.00000005 ETH per gas unit |
| Gas Limit | Maximum gas you're willing to spend | Set to 100,000 for contract interactions |
| Wei | Smallest unit of ETH (10^18 Wei = 1 ETH) | Transfer 1000000000000000000 Wei = 1 ETH |
| Gwei | Billion Wei, commonly used for gas prices | 50 Gwei gas price is typical |
| Solidity | Programming language for Ethereum smart contracts | contract HelloWorld { string public greeting; } |
| ABI | Application Binary Interface - contract function definitions | JSON describing function signatures and types |
| Bytecode | Compiled smart contract code executed by EVM | 0x608060405234801561001057600080fd5b50... |

**Lesson 4 - ERC-20:**
| Term | Definition | Example |
|------|------------|---------|
| ERC-20 | Ethereum standard for fungible tokens | USDT, LINK, UNI are all ERC-20 tokens |
| Token | A digital asset created via smart contract | UNI token represents governance rights in Uniswap |
| Fungible | Interchangeable - each unit is identical | One USDT = any other USDT |
| totalSupply() | Returns total tokens in existence | USDT.totalSupply() = 83,000,000,000 |
| balanceOf() | Returns token balance of an address | token.balanceOf(0x123...) = 1000 |
| transfer() | Send tokens to another address | token.transfer(bob, 100) |
| approve() | Allow another address to spend your tokens | token.approve(uniswap, 1000) |
| allowance() | Check approved spending limit | token.allowance(alice, uniswap) = 1000 |
| transferFrom() | Transfer on behalf of another (requires approval) | uniswap.transferFrom(alice, pool, 100) |
| Minting | Creating new tokens | token.mint(treasury, 1000000) |
| Burning | Destroying tokens permanently | token.burn(100) reduces total supply |

**Complete Implementation for Pausable Feature:**
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";
import "@openzeppelin/contracts/security/Pausable.sol";  // ADD THIS IMPORT

contract StudentToken is ERC20, Ownable, Pausable {  // ADD Pausable inheritance
    uint256 public constant MAX_SUPPLY = 1_000_000 * 10**18;

    constructor() ERC20("StudentToken", "STU") Ownable(msg.sender) {
        _mint(msg.sender, 100_000 * 10**18);  // Initial supply
    }

    // 1. Mint with cap
    function mint(address to, uint256 amount) public onlyOwner {
        require(totalSupply() + amount <= MAX_SUPPLY, "Exceeds max supply");
        _mint(to, amount);
    }

    // 2. Burn tokens
    function burn(uint256 amount) public {
        _burn(msg.sender, amount);
    }

    // 3. Pause functionality
    function pause() public onlyOwner {
        _pause();
    }

    function unpause() public onlyOwner {
        _unpause();
    }

    // 4. Override _update for pause check (OpenZeppelin v5 pattern)
    function _update(address from, address to, uint256 value)
        internal
        override
        whenNotPaused
    {
        super._update(from, to, value);
    }
}
```

**Features to implement:**
1. mint() - owner-only minting with MAX_SUPPLY cap
2. burn() - allow users to burn their own tokens
3. pause()/unpause() - emergency stop with Pausable inheritance
4. Events are inherited from ERC20 (Transfer) and Pausable (Paused/Unpaused)
